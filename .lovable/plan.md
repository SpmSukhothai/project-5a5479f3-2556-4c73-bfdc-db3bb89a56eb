# แผนการแก้ไข 2 หน้า

## 1. หน้า /children — ปวช./ปวส. ประเภท "ราชการ" ไม่ต้องระบุกลุ่มสาขา

ปัจจุบันช่อง "กลุ่มสาขาวิชา" แสดง/บังคับทุกครั้งที่ระดับชั้นเป็น ปวช./ปวส. ไม่ว่าจะเป็นราชการหรือเอกชน และมีทั้ง trigger ในฐานข้อมูลที่บังคับด้วย จะปรับให้บังคับเฉพาะ **เอกชน** เท่านั้น

- เพิ่มตัวช่วย `showsProgramGroup(schoolType, level)` ใน `src/lib/labels.ts` → คืนค่า `true` เฉพาะระดับอาชีวศึกษา (ปวช./ปวส.) ที่เป็น **เอกชน**
- ฟอร์มเพิ่ม/แก้ไขบุตร และฟอร์ม "เปลี่ยนสถานศึกษา": แสดงช่องกลุ่มสาขาเฉพาะเมื่อ `showsProgramGroup` เป็นจริง; ถ้าเป็นราชการให้ล้างค่าและไม่บังคับ
- การตรวจสอบตอนบันทึก (`save` / `submit`): บังคับเลือกกลุ่มสาขาเฉพาะเอกชนอาชีวศึกษา
- ปรับตรรกะ `normalize` ให้ล้าง `program_group_id` เมื่อเป็นราชการ

**ฐานข้อมูล (migration):** แก้ฟังก์ชัน `validate_program_group` ให้บังคับต้องมีกลุ่มสาขาเฉพาะเมื่อ `school_type = 'private'` และเป็นระดับอาชีวศึกษา มิฉะนั้นตั้งค่าเป็นว่าง (NULL) — มีผลกับตาราง children, child_education_history และ reimbursement_rates ที่ใช้ trigger ตัวเดียวกัน

## 2. หน้า /reimbursements — ดึงข้อมูลอัตโนมัติ + วันที่แบบสั้น + ยอดคงเหลือห้ามติดลบ

### 2.1 ดึงข้อมูลอัตโนมัติและล็อก (อ่านอย่างเดียว)
เมื่อเลือกผู้มีสิทธิและบุตรแล้ว ระบบดึง สถานศึกษา / ระดับชั้น / ประเภทโรงเรียน / เงินอุดหนุน / กลุ่มสาขา จากข้อมูลบุตรอยู่แล้ว จะเปลี่ยนช่องเหล่านี้จาก dropdown ที่แก้ได้ ให้เป็น **แสดงผลอย่างเดียว (ล็อก)** เหมือนช่องสถานศึกษา ผู้ใช้ไม่ต้องกรอกเอง

- ระดับชั้น, ประเภทโรงเรียน, เงินอุดหนุน, กลุ่มสาขา → เปลี่ยนเป็นกล่องแสดงข้อความ (read-only) โดยใช้ค่า label ภาษาไทย
- ยังคงคำนวณสิทธิ (rate/entitled) อัตโนมัติจากข้อมูลที่ดึงมา
- ถ้าบุตรยังไม่มีข้อมูล จะแสดงข้อความเตือนให้ไปกรอกที่หน้าบุตรก่อน
- สอดคล้องกับข้อ 1: ถ้าราชการอาชีวศึกษา จะไม่ต้องมีกลุ่มสาขา (ไม่บล็อกการบันทึก)

### 2.2 รูปแบบวันที่แบบสั้น
- เพิ่มฟังก์ชัน `formatThaiDateShort` ใน `src/lib/labels.ts` → รูปแบบ `24 เม.ย. 69` (วัน + เดือนย่อ + ปี พ.ศ. 2 หลัก)
- ใช้ในตารางทะเบียนคุมการเบิก (วันที่จ่าย/วันที่ใบเสร็จ) ให้กระชับขึ้น

### 2.3 ยอดคงเหลือห้ามติดลบ
- ตอนกดบันทึก: ถ้า `สิทธิที่เบิกได้ - (ภาค1 + ภาค2) < 0` → **บล็อกการบันทึก + แจ้งเตือน** ("ยอดเบิกเกินสิทธิที่เบิกได้")
- กล่องสรุป "คงเหลือ" ในฟอร์ม: แสดงเป็นสีแดงเมื่อจะติดลบ เพื่อเตือนผู้ใช้ก่อนบันทึก

## หมายเหตุทางเทคนิค
- ไฟล์ที่แก้: `src/lib/labels.ts`, `src/routes/_authenticated/children.tsx`, `src/routes/_authenticated/reimbursements.tsx` และ migration แก้ฟังก์ชัน `validate_program_group`
- ปัจจุบันมี trigger `trg_children_validate_program_group`, `trg_edu_history_validate_program_group`, `trg_rates_validate_program_group` ที่เรียกฟังก์ชันเดียวกัน จึงแก้ที่ฟังก์ชันครั้งเดียวครอบคลุมทุกฟอร์ม
- คอลัมน์ `program_group_id` เป็น NULL ได้อยู่แล้ว จึงรองรับกรณีราชการที่ไม่มีกลุ่มสาขา
