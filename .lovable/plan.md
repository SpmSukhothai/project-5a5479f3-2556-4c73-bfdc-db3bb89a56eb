## เปลี่ยนเงื่อนไขรหัสผ่าน: อักขระพิเศษเป็นตัวเลือก

### สรุป
ปรับ regex ตรวจสอบรหัสผ่านจาก "บังคับมีอักขระพิเศษ" เป็น "มีได้แต่ไม่บังคับ" โดยยังคงต้องมีตัวหนังสือ + ตัวเลข อย่างน้อย 6 ตัวอักษร

### ไฟล์ที่แก้ไข
1. `src/routes/login.tsx`
   - แก้ `passwordRegex` เป็น: `/^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d!@#$%^&*()_+\-=\[\]{};':"\\|,.<>\/?]{6,}$/`
   - อัปเดต hint text จาก "ประกอบด้วยตัวหนังสือ ตัวเลข และอักขระพิเศษ" เป็น "อย่างน้อย 6 ตัว ประกอบด้วยตัวหนังสือและตัวเลข (อักขระพิเศษเพิ่มได้)"
   - อัปเดต toast error message ให้สอดคล้อง

2. `src/routes/reset-password.tsx`
   - แก้ `passwordRegex` ให้เหมือนกัน
   - อัปเดต hint text และ toast error message ให้สอดคล้อง

### ข้อยกเว้นที่ไม่แก้ไข
- ไม่เปลี่ยนโครงสร้าง UI, ฟอร์ม, หรือ flow การทำงานอื่นๆ
- ไม่เปลี่ยนการตั้งค่า Supabase Auth
