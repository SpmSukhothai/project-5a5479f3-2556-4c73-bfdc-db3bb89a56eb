## เป้าหมาย
ปรับ dialog ฟอร์ม "เพิ่ม/แก้ไขรายการเบิก" ใน `src/routes/_authenticated/reimbursements.tsx` ให้ทันสมัย สะอาดตา ใช้งานง่าย โดยแก้เฉพาะส่วน UI ของฟอร์ม (logic การคำนวณ rate เดิมคงไว้)

## สิ่งที่จะทำ

### 1. ส่วนข้อมูลอัตโนมัติ → กลุ่ม Key–Value / Badge
- แทนที่ช่อง disabled/read-only เดิม (โรงเรียนที่ศึกษา, ระดับชั้น, ประเภทโรงเรียน, เงินอุดหนุน, กลุ่มสาขาวิชา) ด้วยการ์ดข้อมูลสรุป: หัวข้อ (label เล็กสีจาง) + ค่า (ตัวหนา) หรือใช้ `<Badge>` สำหรับ ระดับชั้น/ประเภทโรงเรียน/เงินอุดหนุน
- แสดงเฉพาะเมื่อเลือกบุตรแล้ว; ถ้ายังไม่เลือกให้ขึ้นข้อความชวนเลือกบุตร; กรณีข้อมูลขาด (เช่น ยังไม่มีสถานศึกษา/กลุ่มสาขา) คงข้อความเตือนสีแดงไว้
- เก็บช่องที่ต้องเลือกจริง (ปีการศึกษา, ผู้มีสิทธิ, บุตร) ไว้เป็น input/select ตามเดิม แต่จัดวางให้ชัด

### 2. ภาคเรียนที่ 1 / 2 → `<Tabs>`
- ย้าย 2 กล่องภาคเรียนไปอยู่ใน `<Tabs defaultValue="sem1">` มี `TabsList` 2 แท็บ (ภาคเรียนที่ 1 / ภาคเรียนที่ 2) ค่าเริ่มต้นคือภาคเรียนที่ 1
- แต่ละแท็บใช้ CSS Grid จัดช่องกรอกให้เป็นระเบียบ
- แสดงยอดของแต่ละภาคเรียนบนหัวแท็บหรือในแท็บเพื่อไม่ให้ผู้ใช้ต้องสลับแท็บบ่อย

### 3. ไอคอน + จัดชิดขวา
- เพิ่มไอคอน Lucide ในช่อง input: `Receipt`/`FileText` สำหรับเลขที่เอกสารและเลขที่ใบเสร็จ (ไอคอนซ้อนด้านซ้ายในช่อง)
- ThaiDatePicker มีไอคอนปฏิทินอยู่แล้ว คงไว้
- ช่องจำนวนเงินทุกช่องใช้ `CurrencyInput` ซึ่งจัดตัวเลขชิดขวาอยู่แล้ว — คงไว้และเพิ่มไอคอนสกุลเงิน/ระบุหน่วย "บาท"

### 4. สิทธิที่เบิกได้ = อ่านอย่างเดียว (คำนวณอัตโนมัติ)
- เปลี่ยนช่อง "สิทธิที่เบิกได้" จาก `CurrencyInput` ที่แก้ไขได้ ให้เป็นการแสดงผลอย่างเดียว (ตัวเลขจัดรูปแบบ `formatTHB`) — ผู้ใช้กรอกเองไม่ได้
- ค่ามาจาก `applyRate`/`computeEntitled` เดิม; แสดงคู่กับรูปแบบการเบิก (fixed/percent) ให้ชัด
- ตัด `onChange` ที่ให้ผู้ใช้แก้ `entitled_amount` ออก (ไม่กระทบ logic การคำนวณ)

### 5. การ์ดสรุปยอดเด่นเหนือปุ่ม
- ครอบส่วนสรุป (สิทธิ / ยอดเบิกแล้ว / คงเหลือ) ด้วย `<Card>` พื้นหลังอ่อน (โทน muted/secondary จาก design token — ไม่ hardcode สี) แสดง 3 ค่าเป็นกริดตัวเลขหนาและใหญ่
- คงคำเตือนสีแดงเมื่อยอดเบิกเกินสิทธิ (คงเหลือติดลบ)
- วางการ์ดนี้เหนือ `DialogFooter` (ปุ่มยกเลิก/บันทึก) ให้แยกชัดเจน

### 6. เลย์เอาต์ & Responsive
- ใช้ CSS Grid `grid-cols-1 sm:grid-cols-2` สำหรับฟิลด์ต่าง ๆ ให้สัดส่วนช่องไฟพอดีและ responsive บนมือถือ
- ตรวจ min-w-0/truncate ในการ์ดข้อมูลเพื่อไม่ให้ข้อความล้น

## ขอบเขตทางเทคนิค
