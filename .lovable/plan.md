# แก้ปัญหา: บันทึกเพิ่มอัตราไม่ได้

## สาเหตุ

ตาราง `reimbursement_rates` ยังมีกฎ "ห้ามซ้ำ" แบบเดิมที่ดูแค่ 3 ค่า:

```text
UNIQUE (school_type, education_level, academic_year)
```

ตอนนี้ระบบรองรับอาชีวศึกษาแล้ว อัตราเดียวกันต้องแยกตาม **เงินอุดหนุน (subsidy_type)** และ **กลุ่มสาขาวิชา (program_group_id)** ด้วย แต่กฎเดิมยังไม่นับ 2 ค่านี้ ทำให้เมื่อเพิ่ม "เอกชน + ปวช. + ประมง" มันชนกับแถว ปวช. เดิมที่มีอยู่ จึงขึ้น error `duplicate key`

## วิธีแก้ (เฉพาะฐานข้อมูล)

สร้าง migration เพื่อ:

1. ลบกฎห้ามซ้ำเดิม `reimbursement_rates_school_type_education_level_academic_ye_key`
2. สร้าง unique index ใหม่ที่ครบทั้ง 5 ค่า ตรงกับ logic `findRate` ในโค้ด โดยใช้ค่า sentinel แทน `program_group_id` ที่เป็นค่าว่าง (เพื่อให้แถวที่ไม่ใช่อาชีวะ ยังกันซ้ำได้ถูกต้อง):

```text
UNIQUE (
  school_type,
  subsidy_type,
  education_level,
  COALESCE(program_group_id, '00000000-0000-0000-0000-000000000000'),
  academic_year
)
```

## ผลลัพธ์

- ข้อมูลเดิมทั้งหมดยังอยู่ครบ ไม่ถูกแตะต้อง
- เพิ่มอัตราอาชีวะแยกตามกลุ่มสาขาได้ (เช่น ปวช.+ประมง, ปวช.+พาณิชยกรรม)
- เพิ่มอัตราแยกตามเงินอุดหนุน (รับ/ไม่รับ) ได้
- ยังกันการบันทึกซ้ำของชุดค่าเดิมไว้เหมือนเดิม

ไม่ต้องแก้โค้ด frontend เพราะ `findRate` ใช้คีย์ 5 ค่าตรงกับ index ใหม่อยู่แล้ว
