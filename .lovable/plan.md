# ดึงโรงเรียนของบุตรอัตโนมัติในหน้า /reimbursements

## เป้าหมาย
ในฟอร์ม "เพิ่มรายการเบิก" เมื่อเลือกผู้มีสิทธิ และเลือกบุตรแล้ว ระบบจะดึงข้อมูลสถานศึกษาปัจจุบันของบุตร (ชื่อโรงเรียน + ระดับชั้น + ประเภท) มาแสดงโดยอัตโนมัติแบบอ่านอย่างเดียว และเอาช่องเลือกโรงเรียน (dropdown) ออก

## ปัญหาเดิม
ฟอร์มปัจจุบันให้เลือกโรงเรียนจากรายการแม่ 27 โรง (`schools.school_id`) ซึ่งเป็นโรงเรียนต้นสังกัด ไม่ใช่สถานศึกษาของบุตร ส่วนสถานศึกษาของบุตรเก็บเป็นข้อความอิสระทั่วประเทศ (`children.study_place` / `child_education_history`) จึงไม่ตรงกัน

## สิ่งที่จะทำ

### 1. ปรับโครงสร้างฐานข้อมูล
- เพิ่มคอลัมน์ข้อความ `study_place` ในตาราง `reimbursements` เพื่อเก็บชื่อสถานศึกษาของบุตร (snapshot ณ ตอนเบิก)
- คอลัมน์ `school_id` เดิมคงไว้ (ไม่ลบ) เพื่อความเข้ากันได้ แต่จะไม่ใช้งานต่อ

### 2. ย้ายข้อมูลเก่า
- กรอกค่า `study_place` ของรายการเบิกเดิมที่มี `school_id` โดยดึงชื่อโรงเรียนจากตาราง `schools` มาใส่ให้ครบ

### 3. ปรับหน้าจอ `reimbursements.tsx`
- เอา dropdown "โรงเรียนที่ศึกษา" ออก
- ปรับ query รายชื่อบุตรให้ดึงสถานศึกษาปัจจุบันของบุตร (ชื่อโรงเรียน, ระดับชั้น, ประเภท) จากประวัติการศึกษาที่เป็นปัจจุบัน (`child_education_history` ที่ `is_current = true`)
- เมื่อเลือกบุตร: เติมค่า `study_place`, `education_level`, `school_type` อัตโนมัติ และคำนวณสิทธิที่เบิกได้ (`entitled_amount`) ใหม่จากตารางอัตราตามระดับ/ประเภท/ปีการศึกษา
- แสดงข้อมูลสถานศึกษาเป็นกล่องอ่านอย่างเดียว (ไม่มีตัวเลือก)
- ตอนบันทึก: เก็บลง `study_place` (ข้อความ) แทน `school_id`
- ตารางทะเบียนคุม: แสดงคอลัมน์โรงเรียนจาก `study_place` แทนการ join `schools`
- เอา query รายการโรงเรียน (`s-list2`) และฟังก์ชัน `updateSchool` ที่ไม่ใช้แล้วออก

## หมายเหตุทางเทคนิค
- ระดับชั้น/ประเภท ยังเติมจากข้อมูลบุตรอัตโนมัติ และยังคงแก้ไขได้เพื่อรองรับการคำนวณสิทธิ
- หากบุตรยังไม่มีประวัติการศึกษาปัจจุบัน จะแสดงข้อความแจ้งให้ไปเพิ่มข้อมูลสถานศึกษาของบุตรก่อน
- ไม่แตะตรรกะการคำนวณยอดเบิก/คงเหลือเดิม
